## Goal
CUBRID 개발부 전체(개발자 다수, 기술적 청중)를 대상으로, **김대현(AI TFT, 개발 2팀)이 한 달간 Claude Code Max를 사용해본 현황 보고 + 실용적 Getting-Started 리포트**를 30~40분(발표 20~25분 + Q&A 10분)에 걸쳐 전달한다. 톤은 **"객관적 현황 공유"로 포장된 은근한 advocacy** — 한계/주의점도 정직하게 다루되, 청중이 "나도 한 번 써보자"는 마음이 들 수 있도록 진입 장벽을 낮춘다.

## Constraints
- **발표 시간**: 30~40분 (Q&A 10분 포함, 본 발표 20~25분)
- **청중**: 개발부 전체 (개발 1팀, 2팀, 연구소 등 개발자 다수)
- **기술 수준**: 기술적 청중 — 코드/도구 깊이 있는 설명 가능, 전문 용어 장벽 낮출 필요 없음
- **형식**: 슬라이드 중심 + 사전 녹화 터미널 데모(GIF/영상)
- **안정성 우선**: 라이브 데모 대신 사전 녹화로 부팅/네트워크/API 키 리스크 회피
- **발표자**: 김대현, AI TFT, 개발 2팀
- **요청 주체**: 팀장님, 전무님 (의사결정자 포함 청중)
- **제작 기간**: 1주일 내외 (다음주 수요일 발표)
- **도구 후보**: Marp, Slidev, Keynote, Google Slides 등 (구체 도구는 제작 단계에서 선택)

## Non-Goals
- ❌ 단순 "Claude Code 좋다" 과대광고 피치 (adoption pitch 아님)
- ❌ Claude Code 공식 문서 재포장 / 일반론 나열
- ❌ AI TFT의 메타 워크플로우 자랑 (OMC 등 도구 조합) — 청중과 결이 다름
- ❌ 라이센스/예산 결정 직접 요구 (청중이 스스로 판단)
- ❌ 타 AI 도구(Copilot, Cursor, Codex 등)와의 공격적 비교 — 중립 톤 유지
- ❌ 핸즈온 워크숍 형식 (시간 제약상 불가)
- ❌ 비개발직 청중 대상 용어 단순화 (기술 청중이므로 불필요)
